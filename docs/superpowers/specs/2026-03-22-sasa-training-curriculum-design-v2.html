SASA · AI Training Programme

Curriculum Design Spec

Spec Approved Internal Use 6 Sessions

SASA previously attended a Session 0 intro covering basic AI tools and prompt engineering. That session was hampered by locked machines preventing tool installation. This 6-session programme builds on that foundation, going significantly deeper across coding, data workflows, local AI, and agentic automation.

Audience
Mixed: mill ops, agronomists, analysts, management
Technical Level
Some — Excel-heavy, basic scripting in the room
Format
In-person (<10) + hybrid remote via video link
Session Length
9:30–15:00 (5.5 hours)
Pain points addressed
  • Reporting — Weekly/monthly reports taking too long to compile
  • Data — Working with R7106 mill data (Excel exports, season performance analysis)
  • Communication — Writing reports, summaries, grower letters, proposals
Success criteria
  • Attendees use AI tools independently in day-to-day work
  • Specific people can build scripts, templates, and workflows the organisation actually uses
  • SASA has a clear AI roadmap and a plan to build internal capability
Curriculum

Philosophy & Approach

Project-led learning with largely self-contained sessions. Each session teaches tools in the morning and applies them to a real SASA workflow in the afternoon.

Sessions 1–2 are fully independent. Sessions 3, 4, and 5 form a soft technical track. A pre-built starter script is provided for Sessions 4 and 5 so attendees who missed a prior session can still participate fully.
The SASA Prompt Library (Session 1) is the foundation of the community of practice (Session 6). Establish its location and owner on day one.
Daily session structure
09:30–10:30 Teach — concepts & live demo
10:30–10:45 Break
10:45–12:15 Guided hands-on
12:15–13:00 Lunch
13:00–14:30 Build — apply to real SASA workflow
14:30–15:00 Review + challenge for next session
Session 6 deviates from this structure — no teach block. Morning is showcase/demos, afternoon is strategic roadmap. See Session 6 detail.

Session 1 AI as Your Thinking Partner Deep Dive
Morning — teach
  • Reframe session 0 limitations positively — locked machines, what we missed
  • Mental model shift: AI as junior analyst, not search engine
  • Advanced prompting: role assignment, chain-of-thought, few-shot examples
  • Long document analysis — paste a full mill report, structured questioning
  • Tone, format, audience control — same content, three outputs
  • Microsoft Copilot in Word & Outlook (no install required)
Afternoon — build
  • Each person identifies most time-consuming writing task
  • Build and refine a prompt that produces a usable first draft
  • Collect best prompts into shared SASA Prompt Library (Google Doc / SharePoint)
SASA Prompt Library — 10–15 tested, named prompts for real tasks
Session 2 Excel & Data Mastery with AI Direct Claude Integration
Morning — teach
  • Direct Claude integration with Excel — send data to Claude and receive analysis without leaving the spreadsheet
  • AI-generated formulas — describe what you want, get the formula
  • Data cleaning with Claude — paste messy data, get clean structured output
  • Dashboard building from R7106 exports — pivot tables, conditional formatting via AI
  • Excel Copilot walkthrough (where available)
  • PowerPoint integration — generating slides directly from data. Requires paid Claude account.
Claude Account Recommendation: Full integration features and top-model access require a paid Claude subscription. Recommended: one shared Claude Team or API account at ~$200/month. Unlocks Opus-level models, extended thinking, Projects, and all Excel/PowerPoint integration features used from Session 2 onwards.
Afternoon — build
  • Connect Claude to Excel via the integration layer
  • Work with actual R7106 export data — send to Claude, receive formatted analysis
  • Generate a performance summary and PowerPoint output driven entirely by AI
Live Claude–Excel integration producing automated mill reporting and analysis
Session 3 Coding with AI No Experience Required
Morning — teach
  • New paradigm: describe the problem, AI writes the code, you guide and review
  • Claude Code CLI — install, setup, first commands
  • Windsurf IDE — conversation with your codebase
  • Python basics through AI: reading CSV, filtering, exporting summary
  • When to use Claude Code vs Windsurf vs Claude.ai
Afternoon — build
  • AI-assisted Python script: reads R7106 CSV, calculates per-mill averages, outputs formatted report
  • Everyone runs it on their own machine with real data
Python script — R7106 CSV export → automated summary report
Prerequisite: admin rights + Python 3 pre-installed or IT approval on day
Session 4 Running AI Locally One Machine, Network Access
Morning — teach
  • Why local models matter for regulated / sensitive industry data (mill records, grower data)
  • The network model: Ollama runs as a server on one dedicated machine — all attendees connect via browser, no per-machine setup
  • Installing and configuring Ollama in server mode
  • Model choices: Llama 3, Mistral, Gemma, Phi-3 — trade-offs
  • Open WebUI — all attendees connect via browser like a shared web app
  • Continue.dev extension for VS Code — connects to the local Ollama server
Afternoon — build
  • All attendees connect to shared local AI server via browser
  • Query confidential-style mill report — nothing leaves the network
  • Extend Session 3 Python script to call Ollama server for narrative summaries
  • Starter script provided for attendees who missed Session 3
Shared local AI server accessible to everyone on the network via browser
Intentionally offline session — Claude.ai not used, to demonstrate that capable AI requires no cloud connectivity.
Dedicated machine must be prepared before this session. See hardware spec table below. Model pre-downloaded before session day.

Session 5 Agentic Workflows AI That Does the Work
Morning — teach
  • What agents are: AI + tools + a loop
  • MCP (Model Context Protocol) — connecting Claude to files, Excel, browsers, APIs
  • n8n for no-code automation (open source, self-hostable)
  • Agentic patterns: anomaly flagging, report triggers, grower alerts
Afternoon — build (group chooses one)

All three options use a pre-scaffolded n8n workflow. Attendees configure and extend — not build from scratch. Pre-configured Ollama connection + starter script provided if Session 4 was missed.

  • Option A — Mill Data Monitor: Folder watch → R7106 processing → anomaly-flagged email. Pre-built: folder watch + processor. Attendees configure: credentials, thresholds.
  • Option B — Narrative Report Generator: Mill data → Ollama → SharePoint. Pre-built: data read + Ollama node. Attendees configure: SharePoint target, narrative prompt.
  • Option C — Grower Communication: Data row → letter template → file/email. Pre-built: ingestion + template. Attendees configure: tone, output destination.
Minimum viable outcome: working end-to-end run with real or sample data before the session ends.
Working automated workflow connected to real SASA data source
Session 6 Capstone & Roadmap Build · Demo · Plan
Session 6 deviates from standard structure. No teach block or guided hands-on. Morning = showcase; afternoon = strategic planning.
Morning — showcase (09:30–12:15)
  • 09:30–10:00 — Finish and tidy work from sessions 1–5
  • 10:00–11:15 — Demos: each person/team presents best build (5 min each, up to 10 people). Remote attendees: facilitator to confirm whether presenting or observing — allow extra 10–15 min if presenting.
  • 11:15–11:30 — Break
  • 11:30–12:15 — Group review: what worked, what surprised, what will actually be used
Afternoon — strategic direction (13:00–15:00)
  • AI trajectory in agriculture/agribusiness — 3–5 year view
  • SASA priorities: precision ag, grower analytics, supply chain intelligence, predictive maintenance
  • Internal capability: AI champion role, learning cadence, community of practice
  • Roadmap exercise — each team writes their 90-day AI adoption plan
90-day team action plans + SASA internal AI roadmap
