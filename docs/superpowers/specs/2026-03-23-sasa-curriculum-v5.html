Curriculum Philosophy

How This Programme Works

Core principle: Every session produces a working deliverable using SASA's actual data. No synthetic examples. No slideware deliverables.

Prompt Engineering Thread: Sessions 1–3 carry a deliberate prompt engineering track — embedded in real tasks, not a standalone module. By Session 3, every participant has a personal prompt library, understands context management, and can adapt prompt patterns to new problems independently.

Language discipline: All deliverables framed as "built" or "specified". Invoices reference "Applied AI Consulting" and "integration development" — consistent with Professional Fees budget line.

Sessions 1–3 · Data Foundations + Prompt Literacy

Session Designs

1
Week 1
Foundations Prompt Engineering

Foundations Audit, Strategic Framing & Prompt Foundations

Mapping SASA's data landscape — and building the prompt discipline the whole programme runs on

The first session maps SASA's current data landscape, standardises tooling across the group, and agrees on specific integration targets every subsequent session builds toward. Existing AI work within SASA is reviewed and used as the benchmark for the rest of the group.

Morning — audit & tooling
  • Review AI work already completed by Dave and Sheetal — present as the benchmark, not the exception
  • Map SASA's data landscape: R7106 records, LIMS exports, Autolab CSVs, mill reports — identify highest integration potential
  • Configure tooling: Claude Pro shared account, Copilot licence review, desktop setup for all attendees
  • Define programme capstone goals as a group — integration targets all sessions build toward
Afternoon — Prompt Engineering Foundations (90 min)
  • Prompt anatomy: Role · Context · Task · Format (RCTF) — first template built from real SASA report language
  • Claude-specific techniques: XML tags for structured output; 200K token context window; system prompts; Projects vs. conversations
  • Prompt iteration workflow: draft → test → refine → save — treating prompts as living documents
  • Common failure modes: vague instructions, missing context, unspecified format — diagnosed from real SASA examples
  • Establish SASA shared prompt library in SharePoint: naming conventions, categories, validation discipline
Session Deliverable

SASA data & systems map + integration targets document + consulting log + seed prompt library (10–15 entries)

2
Week 3
Excel Prompt Reuse Copilot

Excel as a Data Engineering Surface

Rebuilding existing Excel workflows as AI-augmented decision tools

Session 2 takes one real SASA Excel report and rebuilds it live using Claude as co-pilot — producing a reusable AI-augmented workflow and exception-flagging template a supervisor can operate independently.

Morning — Excel co-pilot work
  • Rebuild one real SASA Excel report using Claude as live co-pilot
  • Structured data principles: column naming, type consistency, AI-readable format
Prompt Reuse Patterns (45 min)
  • Parameterised templates vs. fixed prompts — when to generalise, when to specialise
  • Excel prompt library: summarise, flag exceptions, extract trends, compare periods
  • Prompt curation and version discipline — avoiding drift across the team
  • Add validated prompts to shared SASA library with naming conventions
Afternoon — Copilot comparison & template build
  • Apply Microsoft Copilot Excel integration to same dataset — compare outputs, establish workflow preference
  • Design exception-flagging template for one reporting area (e.g. lab results outside control limits)
  • Document workflow for supervisor-level operation without technical support
Session Deliverable

Rebuilt AI-augmented Excel report + exception-flagging template + Excel prompt library additions

3
Week 5
RAG Context Management Large Datasets

Working with Large Datasets, RAG & Context Management

Making 10 years of multi-centre SASA data queryable and manageable

Session 3 addresses SASA's scale challenge: how to query 10 years of multi-centre operational data without feeding everything to an AI at once. RAG and context management techniques are introduced and applied to real SASA datasets.

Morning — RAG & data structure
  • Introduce RAG (Retrieval Augmented Generation): the AI-as-librarian pattern — practical analogy, no code required
  • Work with SASA multi-year multi-centre data: slice, filter, summarise using Claude with structured exports
  • Surface data structural problems that limit AI effectiveness
  • Build a simple document index for targeted AI queries
Afternoon — Context Management & Large Dataset Prompting (90 min)
  • Context window management: Claude's 200K token capacity — when to paste vs. chunk vs. use RAG
  • Chunking strategies: split large datasets and query across chunks without losing coherence
  • Summarisation chains: compress 10-year multi-centre data into structured briefings without losing outliers
  • Prompting for synthesis vs. extraction — "what does this mean" vs. "find the anomalies"
  • Multi-document prompting: structuring queries across multiple SASA sources in one conversation
  • Design SASA data-preparation checklist for LIMS, Autolab, and mill formats
Session Deliverable

SASA data-preparation checklist + working RAG demonstration + context management reference card

High-value session for online supervisors — the RAG concept, context guide, and checklist apply across all roles.

Sessions 4–5 · Technical Capability + Reporting Automation

4
Week 7
Python Node.js LIMS Pipeline

Python & Node.js as Integration Glue

Writing and understanding the code that connects SASA's systems to AI

Managers direct and review the code — the AI writes it. This session builds the literacy to do that confidently, then puts it to work on a real LIMS data pipeline.

What we build together
  • Consulting frame: you direct and review — the AI writes the code
  • Build a real Python data pipeline: LIMS export → transformation → flagged output → formatted report
  • Introduce Node.js contrast: Python for data manipulation; Node.js for API/web connectors to SharePoint and source systems
  • Introduction to version control for AI-generated code
  • Each participant identifies one repetitive data task in their area for automation
Session Deliverable

Working LIMS-to-output data pipeline stub + automation candidate list per team area

Prerequisite: Python 3.x, Node.js LTS, and VS Code pre-installed on all attendee machines. IT support required before this session.
5
Week 9
Reporting Copilot Narratives

PowerPoint, Reporting & AI-Generated Narratives

Eliminating manual report commentary and presentation preparation

One of the highest-ROI applications of AI for SASA: replacing the hours spent writing the same interpretive report commentary every cycle with a repeatable, director-approved workflow.

Morning — narrative workflow
  • Audit reporting burden: quantify hours per week writing interpretive text — the ROI conversation for directors
  • Build repeatable narrative workflow: structured data → Claude prompt → draft → human review → sign-off
  • Apply to one actual SASA periodic report
Afternoon — presentation & template build
  • Use Microsoft Copilot PowerPoint integration to generate slides from a SASA data summary
  • Refine via prompt iteration — not just accepting the first output
  • Design report commentary template: SASA house style, regulatory language, exception conventions encoded
  • Produce one complete AI-assisted report draft from real data — reviewed and approved by a director in-session
Session Deliverable

AI-assisted SASA report draft + reusable report commentary prompt template

Session 6 · Programme Close-out

6
Week 11
Local AI Governance Roadmap

Local AI Infrastructure, Governance & Programme Roadmap

Deploying AI inside SASA's environment — and establishing how it is sustained

The final session serves two purposes: deploying SASA's local AI capability so no sensitive data leaves the building, and closing the programme with a governance framework and 12-month roadmap every participant walks away with.

Morning — local AI deployment
  • Open with IT meeting findings — map against Session 3 data boundary decisions
  • Install and configure local AI model on SASA server (with IT present if possible)
  • Walk through Ollama server mode + Open WebUI browser access — no software required on individual machines
  • Design SASA data governance framework: cloud vs. on-premise classification — one-page policy for director sign-off
  • Build local RAG demonstration: index SASA internal documents, query without internet
Afternoon — governance & roadmap close-out
  • Review all six sessions against Session 1 integration targets — what was built, what remains
  • Build SASA AI governance framework: acceptable use policy, data classification, prompt library ownership, model maintenance
  • Design ongoing operating model: who owns what, how new use cases are prioritised, how the consulting log stays current
  • Address skills continuity: advanced-tier roles vs. operational literacy — who needs to go deeper
  • 12-month roadmap: 3–5 integration targets sequenced by value and complexity, with owners and effort
  • Individual 90-day action plans — one concrete next step per participant
Session Deliverable

Local AI server operational + data governance policy + 12-month integration roadmap + individual 90-day action plans

Server minimum: 16 GB RAM · modern CPU · dedicated GPU with 4+ GB VRAM · 50 GB SSD · network-accessible. Recommended: 32 GB RAM · NVIDIA RTX 3060 (12 GB VRAM) · 100 GB SSD.
