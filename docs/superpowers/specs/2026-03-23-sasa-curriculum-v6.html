Curriculum Philosophy

Core principle: Every session produces something usable. The programme is structured so that each session builds on the last — not as abstract theory but as a running consulting engagement with SASA as the client. Participants direct the work; AI assists. The consultant role is to accelerate capability, not to deliver a finished product and leave.

Prompt engineering thread (S1–3): Prompt literacy is the foundation the entire programme rests on. Sessions 1, 2, and 3 each contain a dedicated prompt engineering block that progressively builds skill — from first principles (RCTF) through parameterised templates to RAG-aware context management. Every prompt block adds directly to the shared SASA prompt library on SharePoint, building a reusable institutional asset.

Build day (S6): The final session is deliberately not a lecture. Teams self-select into build groups based on their automation candidates from Session 4, build something real with senior consultant support, and present it. The programme ends with tools on desks and plans in hand — not a certificate on a wall.

Week 1
Foundations Prompt Engineering

Foundations Audit, Strategic Framing & Prompt Engineering

Mapping SASA's data landscape — building the prompt discipline the whole programme runs on

The first session maps SASA's current data landscape, standardises tooling, agrees on integration targets, and establishes the prompt foundations every session builds on.

Morning — Audit & Setup
  • Review advanced members' AI work as benchmark
  • Map SASA data landscape: R7106, LIMS, Autolab, mill reports
  • Configure Claude Pro shared account, Copilot review, setup
  • Define programme capstone goals
Afternoon — Prompt Engineering Block
  • Role · Context · Task · Format (RCTF) — first template from real SASA report language
  • XML tags, 200K context window, system prompts, Projects vs. conversations
  • Iteration workflow: draft → test → refine → save
  • Common failure modes diagnosed from real examples
  • Establish SharePoint prompt library: naming, categories, validation discipline
Deliverable
SASA data & systems map + integration targets + consulting log + seed prompt library (10–15 entries)
2
Week 3
Excel Reporting Copilot

Excel, Reporting & AI-Generated Workflows

Rebuilding data workflows and eliminating manual report commentary — in one day

Two sides of the same problem — turning SASA's data into decisions faster — handled in a single session.

Morning — Excel & Data Workflows
  • Rebuild one SASA Excel report with Claude co-pilot
  • Structured data principles: column naming, type consistency, AI-readable format
Embedded Morning — Prompt Engineering Block
  • Parameterised templates vs fixed prompts
  • Excel prompt library: summarise, flag, extract, compare
  • Version discipline — avoiding drift across the team
  • Add to shared SASA library
Afternoon — Reporting Workflows
  • Audit reporting burden — hours/week on interpretive text — ROI conversation
  • Narrative workflow: structured data → Claude prompt → draft → sign-off
  • Copilot PowerPoint from SASA data summary
  • Report commentary template: house style, regulatory language, exception conventions
  • One complete AI-assisted report draft reviewed by director in-session
Deliverable
AI-augmented Excel report + exception-flagging template + Excel prompt library + report commentary template + completed AI-assisted report draft
3
Week 5
RAG Context Management Large Datasets

Working with Large Datasets, RAG & Context Management

Making 10 years of multi-centre SASA data queryable and manageable

Session 3 addresses SASA's scale challenge — how to query 10 years of operational data without feeding everything to an AI at once.

Morning — RAG Concepts & Data Work
  • Introduce RAG — AI-as-librarian pattern, practical analogy, no code required
  • Work with SASA multi-year multi-centre data: slice, filter, summarise
  • Surface data structural problems that limit AI
  • Build simple document index for targeted queries
Afternoon — Prompt Engineering Block
  • Context window management: Claude's 200K tokens — paste vs chunk vs RAG
  • Chunking strategies: query across chunks without losing coherence
  • Summarisation chains: compress 10-year data without losing outliers
  • Prompting for synthesis vs extraction
  • Multi-document prompting across SASA sources
  • Design SASA data-preparation checklist (LIMS, Autolab, mill formats)
Deliverable
Data-preparation checklist + working RAG demonstration + context management reference card
Note for online supervisors
High-value session for online supervisors — RAG concept, context guide, and checklist apply across all roles.

Week 7
Python Node.js LIMS Pipeline

Python & Automation Pipelines

Writing and understanding the code that connects SASA's systems to AI

Managers direct and review the code — the AI writes it. This session builds the literacy to do that confidently, then puts it to work on a real LIMS data pipeline.

Full Day — What We Build Together
  • Consulting frame: you direct and review, AI writes
  • Python pipeline: LIMS export → transformation → flagged output → formatted report
  • Node.js contrast: Python for data, Node.js for API/SharePoint connectors
  • Version control intro for AI-generated code
  • Each participant identifies one automation candidate — these form Session 6 build briefs
Deliverable
Working LIMS-to-output pipeline stub + automation candidate list per team area (input to Session 6 build day)
⚙ Pre-session requirement: Python 3.x, Node.js LTS, and VS Code pre-installed on all attendee machines. IT support required before this session.
5
Week 9
Local AI Agents Governance

Local AI Infrastructure & Agentic Workflows

The session that changes how SASA thinks about AI — from a tool you ask to a system that acts

This session deploys SASA's local AI capability — keeping sensitive data on-premise — and introduces agentic automation: AI processes that run without constant human instruction.

Morning — Local AI Deployment
  • IT meeting findings mapped against Session 3 data decisions
  • Install and configure Ollama + Open WebUI on SASA server with IT present
  • Browser access for all team members — no software on individual machines
  • Local RAG: index SASA documents, query without internet
  • Design data governance framework: cloud vs on-premise — one-page policy for director sign-off
Afternoon — Agentic Workflows
  • What is an agent? "I ask an AI" vs "the AI does the task and tells me when something needs attention"
  • Introduce n8n: open-source, self-hosted, no per-seat licence
  • Build SASA agent pipeline: data → exception check → draft alert → SharePoint/file output
  • Connect to Session 4 Python output — real data, not a demo
  • Design SASA's first agentic use case specification: inputs, triggers, outputs, human review gate
Deliverable
Local AI server operational + working n8n agent pipeline + agentic use case specification + data governance policy

Week 11
Build Day Governance Roadmap

Build Day, Governance & Programme Roadmap

Teams build something real — the programme closes with a clear path forward

The final session is structured in two halves: a morning build day where teams construct a working tool, and an afternoon that turns the programme into a sustained 12-month capability.

Morning — Build Day
  • Teams self-select into 2–3 build groups from Session 4 automation candidate lists
  • Build targets: Excel exception report / Python data pipeline / local RAG / n8n agent workflow
  • Richard rotates as senior consultant; advanced members lead their groups
  • GOAL: every group ships something usable from Monday — not a prototype
Afternoon — Governance & Roadmap Close-out
  • Showcase: each group presents their build (5 minutes each)
  • Review all sessions against Session 1 integration targets — what was built, what remains
  • Governance framework: acceptable use, data classification, prompt library ownership
  • Operating model: who owns what, how new use cases are prioritised
  • Skills continuity: who deepens, who needs operational literacy
  • 12-month roadmap: 3–5 targets with owners and effort
  • Individual 90-day action plans
Deliverable
Team-built working tools (one per group, usable immediately) + data governance policy + 12-month integration roadmap + individual 90-day action plans
