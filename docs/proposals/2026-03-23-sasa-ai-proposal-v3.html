— Executive Summary

A Professional Services Engagement

This proposal outlines a six-session Applied AI Consulting and Development Programme delivered at SASA's Mount Edgecombe facility. This is a professional services engagement under the professional fees budget — not a training initiative.

The focus throughout is on building, integrating, and deploying AI-driven tools and workflows directly into SASA's existing systems and data environment — using SASA's own operational data and building directly on the foundations already established by members of the team. Every session produces a working deliverable SASA retains and uses immediately. The programme concludes with a working local AI deployment, a data governance policy, and a 12-month integration roadmap.

— Approach

How We Work Together

Every session works with SASA's actual data and systems. The deliverables are working tools, tested integrations, documented workflows, and deployable pipelines.

  • We build using company data, not constructed examples — every exercise uses real SASA datasets
  • Each session produces a tangible deliverable SASA keeps and can use from the following Monday
  • Advanced team members are leveraged as context contributors and peer demonstrators — not held back
  • Sessions accommodate different capability levels simultaneously — some building, others directing and reviewing
  • All co-developed assets (scripts, prompt libraries, specifications) are stored in a shared SASA consulting log accessible to all participants including online attendees
  • Sessions can be paused to consolidate, catch up, or re-focus based on team progress and priorities

— Session Details

What Happens Each Day

Session 1

Foundations Audit, Strategic Framing & Prompt Foundations Mapping SASA's data landscape — and building the prompt discipline the whole programme runs on

The first session maps SASA's current data landscape, standardises tooling, agrees on integration targets, and establishes the prompt foundations every session builds on. Existing AI work within SASA is reviewed and used as the benchmark for the rest of the group.

Morning — audit & tooling
  • Review AI work already completed by advanced team members — present as the benchmark, not the exception
  • Map SASA's data landscape: R7106 records, LIMS exports, Autolab CSVs, mill reports — identify highest integration potential
  • Configure tooling: Claude Pro shared account, Copilot licence review, desktop setup for all attendees
  • Define programme capstone goals as a group — integration targets all sessions build toward
Afternoon — Prompt Engineering Foundations
  • Prompt anatomy: role, context, task, format (RCTF) — first template built from real SASA report language
  • Claude-specific techniques: XML tags, 200K context window, system prompts, Projects vs. conversations
  • Prompt iteration workflow: draft → test → refine → save as living documents
  • Common failure modes: vague instructions, missing context, unspecified format — diagnosed from real examples
  • Establish SASA shared prompt library in SharePoint: naming, categories, validation discipline
Session Deliverable

SASA data & systems map + integration targets document + consulting log + seed prompt library (10–15 entries)

Session 2

Excel as a Data Engineering Surface Rebuilding existing Excel workflows as AI-augmented decision tools

Session 2 takes one real SASA Excel report and rebuilds it live using Claude as co-pilot — producing a reusable AI-augmented workflow and exception-flagging template a supervisor can operate independently.

Morning — Excel co-pilot work
  • Rebuild one real SASA Excel report using Claude as a live co-pilot
  • Structured data principles: column naming, type consistency, AI-readable format
Prompt Reuse Patterns (45 min)
  • Parameterised templates vs. fixed prompts — when to generalise, when to specialise
  • Excel prompt library: summarise, flag exceptions, extract trends, compare periods
  • Prompt curation and version discipline — avoiding drift across the team
  • Add validated prompts to shared SASA library with naming conventions
Afternoon — Copilot comparison & template build
  • Apply Microsoft Copilot Excel integration to same dataset — compare outputs, establish workflow preference
  • Design exception-flagging template for one reporting area (e.g. lab results outside control limits)
  • Document workflow for supervisor-level operation without technical support
Session Deliverable

Rebuilt AI-augmented Excel report + exception-flagging template + Excel prompt library additions

Session 3

Working with Large Datasets, RAG & Context Management Making 10 years of multi-centre SASA data queryable and manageable

Session 3 addresses the core challenge of SASA's scale: how do you query 10 years of multi-centre operational data without feeding everything to an AI at once? Retrieval Augmented Generation and context management techniques are introduced and applied to real SASA datasets.

Morning — RAG & data structure
  • Introduce RAG (Retrieval Augmented Generation): the AI-as-librarian pattern — practical analogy, no code required
  • Work with SASA multi-year multi-centre data: slice, filter, summarise using Claude with structured exports
  • Surface data structural problems that limit AI effectiveness
  • Build a simple document index for targeted AI queries
Afternoon — Context Management & Large Dataset Prompting
  • Context window management: Claude's 200K token capacity — when to paste vs. chunk vs. use RAG
  • Chunking strategies: split a large dataset and query across chunks without losing coherence
  • Summarisation chains: compress 10-year multi-centre data into structured briefings without losing outliers
  • Prompting for synthesis vs. extraction — "what does this mean" vs. "find the anomalies"
  • Multi-document prompting: structuring queries across multiple SASA sources in one conversation
  • Design SASA data-preparation checklist for LIMS, Autolab, and mill formats
Session Deliverable

SASA data-preparation checklist + working RAG demonstration + context management reference card

High-value session for online supervisors — the RAG concept, context guide, and checklist apply across all roles.
Session 4

Python & Node.js as Integration Glue Writing and understanding the code that connects SASA's systems to AI

Session 4 is a capability-building session. Managers direct and review the code — the AI writes it. The session builds the literacy to do that confidently, then puts it to work on a real LIMS data pipeline.

What we build together
  • Consulting frame established: you direct and review — the AI writes the code
  • Build a real Python data pipeline: LIMS export → transformation → flagged output → formatted report
  • Introduce Node.js contrast: Python for data manipulation; Node.js for API/web connectors to SharePoint and source systems
  • Introduction to version control for AI-generated code
  • Each participant identifies one repetitive data task in their area for automation
Session Deliverable

Working LIMS-to-output data pipeline stub + automation candidate list per team area

Prerequisite: Python 3.x, Node.js LTS, and VS Code pre-installed on all attendee machines. IT support required before this session.

Session 5

PowerPoint, Reporting & AI-Generated Narratives Eliminating manual report commentary and presentation preparation

One of the highest-ROI applications of AI for SASA: replacing the hours spent writing the same interpretive report commentary every cycle with a repeatable, director-approved workflow.

Morning — narrative workflow
  • Audit the current reporting burden: quantify hours per week writing interpretive text — the ROI conversation
  • Build a repeatable narrative workflow: structured data → Claude prompt → draft → human review → sign-off
  • Apply to one actual SASA periodic report
Afternoon — presentation & template build
  • Use Microsoft Copilot PowerPoint integration to generate slides from a SASA data summary
  • Refine via prompt iteration — not just accepting the first output
  • Design report commentary template: SASA house style, regulatory language, exception conventions encoded
  • Produce one complete AI-assisted report draft from real data — reviewed and approved by a director in-session
Session Deliverable

AI-assisted SASA report draft + reusable report commentary prompt template

Session 6

Local AI Infrastructure, Governance & Programme Roadmap Deploying AI inside SASA's environment — and establishing how it is sustained

The final session serves two purposes: deploying SASA's local AI capability so no sensitive data leaves the building, and closing the programme with a governance framework and a 12-month roadmap every participant walks away with.

Morning — local AI deployment
  • Open with IT meeting findings — map against Session 3 data boundary decisions
  • Install and configure local AI model on SASA server (with IT present if possible)
  • Walk through Ollama server mode + Open WebUI browser access — no software required on individual machines
  • Design SASA data governance framework: cloud vs. on-premise classification — one-page policy for director sign-off
  • Build local RAG demonstration: index SASA internal documents, query without internet
Afternoon — governance & roadmap close-out
  • Review all six sessions against Session 1 integration targets — what was built, what remains
  • Build SASA AI governance framework: acceptable use policy, data classification, prompt library ownership, model maintenance
  • Design ongoing operating model: who owns what, how new use cases are prioritised, how the consulting log stays current
  • Address skills continuity: advanced-tier roles vs. operational literacy — who needs to go deeper
  • 12-month roadmap: 3–5 integration targets sequenced by value and complexity, with owners and effort
  • Individual 90-day action plans — one concrete next step per participant
Session Deliverable

Local AI server operational + data governance policy + 12-month integration roadmap + individual 90-day action plans

Server minimum: 16 GB RAM, modern multi-core CPU, dedicated GPU with 4+ GB VRAM, 50 GB SSD, network-accessible. Full specification in Technical Requirements.

— Reference

AI Terminology

Plain-language definitions for terms used throughout this proposal.

TermPlain-language definition
AISoftware that understands language, analyses data, and generates text or code — without being programmed with explicit rules for every situation
LLMThe type of AI powering Claude, ChatGPT, and Copilot — trained on large volumes of text to understand and generate language
ClaudeAnthropic's AI assistant — the primary tool used in this programme. Available via browser, API, and mobile
Claude ProA paid subscription tier (~$100/month) giving access to the most capable models and higher usage limits
PromptThe instruction or question you give to an AI. How you phrase it significantly affects the quality of the response
Prompt EngineeringThe practice of crafting prompts precisely to get reliable, high-quality outputs from an AI model
Context WindowHow much text an AI can "see" at once. Claude supports up to 200,000 tokens — enough for very large documents and datasets
RAGRetrieval Augmented Generation — AI retrieves relevant information from a document store before responding, enabling answers about large or private datasets
TokenThe unit an AI processes — roughly one word or part of a word. Pricing and context limits are measured in tokens
CopilotMicrosoft's AI integration built into Microsoft 365 (Word, Excel, PowerPoint, Outlook). SASA already holds licences
APIApplication Programming Interface — a structured way for software systems to talk to each other, used to connect Claude to Excel, LIMS, or other systems
PythonA widely used programming language suited to data manipulation and automation. All code in this programme is AI-generated — no prior experience required
Node.jsA JavaScript-based environment for building web-facing connectors and API integrations
VS CodeA free code editor from Microsoft, used to review, run, and manage AI-generated scripts
OllamaA free, open-source tool for running AI models locally on a server — no internet or cloud service required
Open WebUIA browser-based interface for a locally running Ollama model — looks like ChatGPT but runs on SASA's own hardware
Agentic WorkflowAn AI set up to complete multi-step tasks autonomously — e.g. check data, detect an exception, draft an alert, send it — without a human prompting each step
n8nAn open-source, self-hostable workflow automation tool used to build agentic pipelines — free, runs on SASA's own infrastructure
LIMSLaboratory Information Management System — SASA's laboratory data management system, a key integration target
AutolabSASA's laboratory instrument automation system — feeds data into LIMS; an upstream integration point for anomaly detection
