01 — Overview

Executive Summary

This proposal outlines a structured six-session Applied AI programme for the South African Sugar Association. Over approximately eleven weeks, your teams will move from AI awareness to operational capability — producing real outputs in each session, not just attending presentations.

The programme is built around SASA's actual data systems: R7106, LIMS, Autolab, and mill reporting workflows. Every session is grounded in real problems, real documents, and real outputs. By the final session, teams will have built and shipped at least one working tool.

02 — Method

Approach

This is not a training course. It is a consulting engagement where learning happens through doing. The principles below govern how every session is designed and delivered.

  • Grounded in your data. Every exercise uses SASA's real systems and real documents — not synthetic examples or generic datasets.
  • Output in every session. Each session ends with a tangible deliverable that SASA staff can use from the following Monday.
  • Managers direct, AI writes. The programme builds prompting and oversight skills, not dependency. Your team learns to use AI as a junior analyst, not a black box.
  • Zero-cost tool stack. Where possible, all tools are open-source and self-hosted (Python, n8n, Ollama, Open WebUI). No per-seat licensing beyond Claude Pro.
  • Build, don't present. The final session is a live build day. Teams ship working tools, not slide decks.
  • Governance from day one. Data classification, acceptable use, and prompt ownership are introduced in Session 1 and formalised in Session 6.
  • Senior consultant throughout. Richard leads all sessions. Advanced participants co-facilitate in Session 6 — building internal capability, not dependency on the consultant.

09 — Reference

Glossary of Terms

For participants new to AI tooling, this glossary defines the key terms used throughout this proposal and the programme.

Term Definition
AI Artificial Intelligence — software systems that perform tasks typically requiring human reasoning, such as reading, summarising, and generating text.
LLM Large Language Model — the class of AI model underlying tools like Claude and ChatGPT. Trained on vast text corpora; responds to natural language instructions.
Claude Anthropic's large language model. Noted for long context windows, instruction-following accuracy, and safety. Used as the primary AI tool in this programme.
Claude Pro The paid subscription tier of Claude (~R1,700/month). Provides expanded context, higher usage limits, and shared workspace functionality.
Prompt The instruction or question given to an AI model. Quality of output is directly determined by quality of the prompt.
Prompt Engineering The craft of writing structured, effective prompts to get reliable, accurate, and useful responses from an AI model.
Context Window The maximum amount of text an AI model can read and reason over in a single interaction. Claude's context window is approximately 200,000 tokens (~150,000 words).
RAG Retrieval-Augmented Generation — a technique where relevant documents are retrieved from a database and inserted into the prompt, enabling AI to answer questions about large document sets without fine-tuning.
Token The basic unit of text processed by an LLM. Roughly 4 characters or ¾ of a word. Context windows and costs are measured in tokens.
Copilot Microsoft's AI assistant integrated into Office 365 (Word, Excel, PowerPoint, Teams). Complements Claude for document and spreadsheet workflows.
API Application Programming Interface — a defined connection point that allows two software systems to communicate. Used to connect Python pipelines to LIMS and SharePoint.
Python A free, widely-used programming language favoured for data processing and automation. AI can write Python code; managers direct the logic.
Node.js A JavaScript runtime used for building API connectors and web-based tools. Used in this programme for SharePoint integration.
VS Code Visual Studio Code — a free, industry-standard code editor. Used to view, edit, and run AI-generated Python and Node.js code.
Ollama A free, open-source tool that runs large language models locally on a server. Enables SASA to use AI on internal documents without sending data to the cloud.
Open WebUI A browser-based interface for Ollama. Allows all participants to access the local AI model via a web browser — no software installed on individual machines.
Agent / Agentic Workflow An AI system configured to take a sequence of actions autonomously — not just answer a question, but execute a multi-step task (e.g., check data → draft alert → send to SharePoint).
n8n An open-source, self-hosted workflow automation tool. Used to build SASA's agent pipelines without per-seat licensing costs. Connects AI outputs to internal systems.
LIMS Laboratory Information Management System — SASA's system for recording and managing laboratory sample data, including quality and compliance records.
Autolab SASA's automated laboratory analysis system. Produces structured data outputs that are used as inputs to AI pipelines in this programme.
