## Programme at a Glance

### Session 1 — Foundations Audit & Prompt Foundations
- Map SASA data landscape (R7106, LIMS, Autolab, mill reports)
- Configure tooling and shared Claude account
- Prompt engineering fundamentals: structure, iteration, Claude-specific techniques
- Establish shared prompt library and consulting log

### Session 2 — Excel as a Data Engineering Surface
- Rebuild a SASA Excel report using Claude as co-pilot
- Build reusable Excel prompt library (summarise, flag, extract, compare)
- Design exception-flagging template for one reporting area
- Copilot Excel integration — compare with Claude workflow

### Session 3 — Large Datasets, RAG & Context Management
- Retrieval Augmented Generation: query large SASA datasets without uploading everything
- Context window management: when to paste, chunk, or use RAG
- Chunking and summarisation chains for 10-year multi-centre data
- SASA data-preparation checklist for LIMS, Autolab, and mill formats

### Session 4 — Python & Node.js as Integration Glue
- AI-generated Python pipeline: LIMS export → transformation → flagged output
- Node.js for API and SharePoint connectors
- Version control basics for AI-generated code
- Automation candidate list per team area (inputs for Sessions 7–8)

### Session 5 — Reporting & AI-Generated Narratives
- Eliminate manual report commentary with repeatable Claude workflows
- AI-assisted report draft from real SASA data — reviewed by a director in-session
- Copilot PowerPoint: generate slides from a data summary
- Reusable report commentary template in SASA house style

### Session 6 — Running AI on SASA Infrastructure
- Install local AI model on SASA server (Ollama + Open WebUI)
- Browser access for all team members — no software on individual machines
- SASA data governance policy: cloud vs. on-premise classification
- Local RAG: query SASA documents without internet

### Session 7 — LIMS & Autolab Integration
- Map LIMS data flow; define exception business rules with lab managers
- Build AI-assisted exception detection layer (Python or prompt chain)
- Notification architecture: flag → alert → channel
- Autolab upstream integration specification

### Session 8 — Agentic Workflows & Automation Pipelines
- Build multi-step agentic workflow: data → AI check → exception → alert → human review
- n8n automation pipelines — pre-built scaffold, configure and extend
- Failure modes, human override, and audit trail design
- Each participant maps one workflow for full automation

### Session 9 — Source System Integration & Deployment
- Build functional integration prototypes from Session 8 maps
- API connector pattern (Node.js/Python): AI layer ↔ source system, no manual exports
- Autolab-to-LIMS chain review and partial live implementation
- SASA AI architecture diagram + deployment checklist

### Session 10 — Capstone, Governance & Roadmap
- Live demos of all integrations built across the programme (directors present)
- Structured review against Session 1 targets
- SASA AI governance framework: acceptable use, data classification, prompt library ownership
- 12-month integration roadmap + individual 90-day action plans
